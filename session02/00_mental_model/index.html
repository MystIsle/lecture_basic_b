멘탈 모델 3단계

각 단계를 클릭하면 특징이 펼쳐집니다

Stage 1
따라 치는 사람
코드 복붙, 돌아가면 OK
예시: 튜토리얼 따라하기, 에러나면 구글링
  • 코드를 이해하지 않고 복사-붙여넣기로 진행
  • 컴파일 에러 메시지를 읽기보다 구글링
  • 왜 이 코드를 써야 하는지 묻지 않음
  • 프로젝트마다 같은 부분에서 같은 실수 반복
반복 경험 — 여러 프로젝트를 만들어보기
Stage 2
도구를 쓸 줄 아는 사람
CMC, BP 노드 조합. "이걸 쓰면 된다"를 안다
예시: AddMovementInput 호출, 블루프린트 조합
  • 공식 문서를 통해 API와 도구를 검색할 수 있음
  • "이 기능을 만들려면 X 함수를 써야 한다"를 인식
  • 기본 논리 흐름(Input → Logic → Output)을 이해
  • 다양한 프로젝트에 도구 조합을 적용 가능
✓ 대부분의 개발자는 이 단계에서 생산성이 가장 높습니다
원리를 직접 만져보기 — 도구를 벗기고 밑바닥을 보는 경험
Stage 3
원리를 아는 사람
"이 안에서 뭘 하고 있는지" 읽힘. 공식 문서가 읽힌다
예시: DeltaTime 왜 곱하는지, CMC 내부 로직
  • API 내부의 구현 원리를 추적하고 이해
  • 공식 문서의 한 줄짜리 설명이 무엇을 의미하는지 파악
  • 도구 선택의 trade-off를 이해하고 평가
  • 예상 밖의 버그를 체계적으로 진단 가능
💡 3단계는 도달 목표가 아니라, 2단계를 더 잘하기 위한 보조 무기입니다
막힐 때 구글링 키워드가 달라집니다
1단계: "C++ 에러 메시지" → 2단계: "언리얼 API 함수명" → 3단계: "원리와 내부 동작"
